## 是什么

Memvid 是一个**可移植的 AI 记忆系统**，受视频编码启发，将数据、embedding、检索结构和元数据打包进**单个文件**（类似视频帧），无需运行任何服务器或向量数据库。

核心抽象是 **Smart Frame**（智能帧）：每帧是不可变的内容单元，包含内容、时间戳、校验和和元数据。帧按组压缩，支持高效并行读取。

## 核心特性

| 特性 | 说明 |
|------|------|
| **单文件架构** | 整个记忆库就是一个文件，随身携带，无需服务器 |
| **Append-only** | 只追加写入，不修改历史帧，天然崩溃安全 |
| **时序查询** | 支持查询过去任意时间点的记忆状态 |
| **知识演化追踪** | Timeline 式检查知识如何随时间变化 |
| **模型无关** | 不绑定任何 LLM，任意 Agent 框架可接入 |
| **无基础设施依赖** | 不需要 Qdrant、Pinecone、Weaviate 等向量数据库 |

## Smart Frame 设计

受视频编码启发，但目的不是存储视频，而是将 AI 记忆组织为**追加式、超高效的帧序列**：

- 帧不可变 → 天然崩溃安全
- 帧分组压缩 → 高压缩率
- 并行读取 → 超低延迟
- 时间戳 → 支持历史查询

## 与传统向量数据库的对比

| 维度 | 向量数据库（Qdrant等） | memvid |
|------|----------------------|--------|
| 部署 | 需要独立服务 | 单文件，零部署 |
| 可移植性 | 低（数据在服务器） | 极高（复制文件即迁移） |
| 检索延迟 | 毫秒级 | 0.025ms P50 |
| 历史查询 | 不支持 | 支持任意时间点 |
| 基础设施成本 | 高 | 零 |

## 适合场景

- 需要给 Agent 添加长期记忆但不想维护数据库基础设施
- 边缘部署、离线场景的 AI 应用
- 需要记忆可移植（换机器/换环境）的个人 AI 助手
- 对检索延迟要求极高的实时 Agent

## 注意事项

- Rust 实现，Python binding 通过 `memvid-core` crate 接入
- 单文件架构在超大规模（TB 级）场景下需要评估分片策略
- 目前主要面向长对话记忆场景，知识图谱类需求支持有限
