## 是什么

MetaGPT 是最早将"角色扮演多智能体"理念产品化的开源框架。它把软件开发流程拆解为多个专职 Agent 的协作：产品经理分析需求、架构师设计方案、工程师编写代码、QA 写测试用例，像一家真实的软件公司一样协作完成任务。

2023 年发布后迅速成为多 Agent 领域标杆，被引用于大量学术研究和工业落地项目。

## 核心架构

| 角色 | 职责 |
|------|------|
| **ProductManager** | 需求分析，产出 PRD |
| **Architect** | 系统设计，生成技术方案 |
| **Engineer** | 代码实现 |
| **QaEngineer** | 测试用例编写与验证 |
| **Environment** | 共享消息黑板，Agent 间通信 |

## 核心功能

- **SOP（标准作业程序）**：用结构化流程约束 Agent 行为，避免幻觉漂移
- **角色继承体系**：内置多种角色可直接扩展，也可自定义新角色
- **共享消息池**：所有 Agent 通过黑板模式通信，无需点对点传递
- **代码执行验证**：生成代码后自动运行测试，形成闭环
- **增量开发**：支持在已有代码库上增量迭代，而非每次从零生成

## 典型用法

```bash
pip install metagpt
metagpt "Write a cli blackjack game"
```

## 适合场景

- 需要从需求一键生成完整软件项目的团队
- 多智能体学术研究的基础框架
- 希望用 LLM 模拟复杂组织协作的探索性项目

## 注意事项

- SOP 流程较重，灵活性低于轻量框架
- 复杂任务 token 消耗大，建议搭配 prompt caching
- 生成代码质量依赖底层模型能力，需人工 review
